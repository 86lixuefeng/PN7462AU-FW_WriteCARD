§ phRtos_TaskState_t

enum phRtos_TaskState_t

Indicates the current state of the Task.

Enumerator
eTaskRunning 

A task is querying the state of itself, so must be running.

eTaskReady 

The task being queried is in a read or pending ready list.

eTaskBlocked 

The task being queried is in the Blocked state.

eTaskSuspended 

Task being queried is in Suspended OR in Blocked state with an infinite time out.

eTaskDeleted 

The task being queried has been deleted.

§ phRtos_EventGroupWaitBits()

uint32_t phRtos_EventGroupWaitBits ( phRtos_EventHandle_t  eventHandle,
const uint32_t  dwBitsToWait,
const bool  bClearOnExit,
const bool  bWaitForAllBits,
uint32_t  dwBlockTime 
)

Wait for one or more Events/Signals Flags to become signaled for the specified event group.

Parameters
[in]eventHandleThe event group in which the bits are being tested.
[in]dwBitsToWaitA bitwise value that indicates the bit or bits to test inside the event group.
[in]bClearOnExitIf true then any bits within uxBitsToWait that are set will be cleared. If false then the bits set in the event group are not altered.
[in]bWaitForAllBitsIf true then wait until all specified event flags set else for any single flag.
[in]dwBlockTimeThe time in ticks to wait for one/all (depending on the xWaitForAllBits value) of the bits specified by uxBitsToWaitFor to become set.
Returns
Returns the value of the event group at the time either the bits being waited for became set, or the block time expired.

bool  bAutoReload,
pphRtos_TimerCallback_t  fpCallbackFunction 
)

Creates a new timer instance.

Returns a handle by which the new timer can be referenced.

Parameters
dwTimerID: User assigned timer Id.
dwTimePeriodInTicks: The timer period in Ticks.
bAutoReload: If set to pdTRUE then the timer will expire repeatedly with a frequency set by the xTimerPeriodInTicks parameter. If uxAutoReload is set to pdFALSE then the timer will be a one-shot timer.
fpCallbackFunction: The function to call when the timer expires.
Returns
If the timer is successfully created then a handle to the newly created timer is returned. If the timer cannot be created then NULL is returned.

Parameters
timerHandle: The handle of the timer being stopped.
dwBlockTime: Specifies the time in ticks that the calling task should be held in the Blocked state to wait for the stop command to be successful.
Returns
phRtos_Failure will be returned if the stop command could not be success even after dwBlockTime ticks had passed. phRtos_Success will be returned if the command is successful.
