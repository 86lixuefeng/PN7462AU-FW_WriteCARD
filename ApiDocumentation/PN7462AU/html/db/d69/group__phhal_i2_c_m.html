Detailed Description

The I2C Master APIs.

This module implements I2CM interface Flash code HAL, specific to I2CM module. This module is not designed for multi thread usage.

Warning

If the build Macro PHFL_PARAM_CHECK is less than PH_PARAM_CHECK_VERBOSE, then the input parameters will not be checked for erroneous values in the HAL API. In such a build configuration use the API with caution and pass valid parameters only. Invalid parameters will likely result in hardfaults or other failures.

§ phhalI2CM_SlaveCheck()

phStatus_t phhalI2CM_SlaveCheck ( uint16_t  wSlaveAddr)

This API Performs I2C slave presence check on I2C bus for a specific 7 bit slave address.

Note
This API shall be called ( may be multiple times) for different slaves before performing transactions(Queue/Read/Write) to these slaves.
Calling this API in between any transaction may result in undefined behavior.
Parameters
[in]wSlaveAddr7 bit Address of the slave to be checked
Returns
Status of the Operation
PH_ERR_SUCCESS If the operation completed successfully, slave presence is detected.
Return values
maskedPH_ERR_INVALID_PARAMETER Invalid passing parameter error.
maskedPH_ERR_TXRX_STARTED_ERROR I2C Master transaction ongoing error.
maskedPH_ERR_OPERATION_TIMEDOUT Operation didn't succeeded wit in specified dwTicksToWait time.
maskedPH_ERR_ADDR_NAK_ERROR I2C Master returned with NAK (either slave is busy/not present).
maskedPH_ERR_ARB_ERROR I2C Master looses the bus.
maskedPH_ERR_DATA_NAK_ERROR Slave naked the data reception.
maskedPH_ERR_DATA_BUS_ERROR I2C bus error.
maskedPH_ERR_QUEUE_NOT_EMPTY_ERROR I2C queue is already filled with other Tx/Rx transactions.
