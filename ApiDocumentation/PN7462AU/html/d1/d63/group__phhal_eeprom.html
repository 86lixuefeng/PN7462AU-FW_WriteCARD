Detailed Description

The EEPROM Controller Read/Write APIs.

Warning
It is impossible to have more than one write or read of EEPROM at any moment of time.
This module uses the phUser_Wait() API. Hence the user is expected to initialise this mechanism by calling phUser_WaitInit() before calling any API from this module.
If the build Macro PHFL_PARAM_CHECK is less than PH_PARAM_CHECK_VERBOSE, then the input parameters will not be checked for erroneous values in the HAL API. In such a build configuration use the API with caution and pass valid parameters only. Invalid parameters will likely result in hardfaults or other failures.

Macro Definition Documentation

§ phhalEeprom_Init

#define phhalEeprom_Init (   bEnableFastMode)

This function initializes the EEPROM HAL.

Warning
The EEPROM memory is Powered up. Normally this takes 100us. The user must not perform any EEPROM access for a period of 100us after calling the Init API of EEPROM HAL.
Note
This Function must be called once before using other APIs in this module.
This EEPROM HAL uses the phUser_Wait() API. Hence the user is expected to initialise this mechanism by calling phUser_WaitInit() before calling phhalEeprom_Init().
Parameters
[in]bEnableFastMode8bit unsigned integer used as Boolean value to enable or disable Fast mode for EEPROM access. In Fast mode, there are less wait states during the Write and read access operations to EEPROM. However, there is no change in the duration of the program phase for write operations.
Returns
Status of the Operation
Return values
PH_ERR_SUCCESSalways

§ phhalEeprom_ReadBuffer

#define phhalEeprom_ReadBuffer (   pbBuffer,
  pbEeprom,
  dwBytesToRead 
)

This function reads EEPROM data.

EEPROM memory can be read like RAM, using pointers. The value that this API adds is that it takes care of avoiding memory access violations, even when one or both of the addresses are mis-aligned. Additionally, parameter checking is also done, if enabled in the build.

Note
This function reads EEPROM data without ECC error reporting. Upper Software layer must take care of validating the read data, if desired.
Parameters
[out]pbBufferPointer to buffer where data will be copied
[in]pbEepromPointer to EEPROM address to be read
[in]dwBytesToReadNumber of bytes to be read. Must not be zero.
Returns
Status of the Operation
Return values
PH_ERR_SUCCESSif EEPROM Read Operation completed successfully(Error checking not done by the API).
maskedPH_ERR_INVALID_PARAMETER if parameters passed are invalid and parameter checking is enabled in build.

§ phhalEeprom_WritePage

#define phhalEeprom_WritePage (   dwPageNumber,
  pbBuffer 
)

This function writes a single full 64byte EEPROM page, with data provided by the user.

Note
EEPROM pages are numbered from 0 to 63, but in User mode, EEPROM pages that can be used range from "PHHALHW_EEPROM_USERPAGE_START" to "PHHALHW_EEPROM_USERPAGE_END"
The Write is synchronous i.e.Polling based. The API will return only after the write operation completes.
Parameters
[in]dwPageNumberValid index of a 64byte page in the EEPROM
[in]pbBufferPointer to buffer containing data to be written to EEPROM. Must be 4 byte aligned
Returns
Status of the Operation
Return values
PH_ERR_SUCCESSif EEPROM Write Operation completed successfully.
maskedPH_ERR_READ_WRITE_ERROR HW related errors.
maskedPH_ERR_INVALID_PARAMETER if parameters passed are invalid and parameter checking is enabled in build.
maskedPHHAL_ERR_EEPROM_BUSY if another EEPROM write is already ongoing.

§ phhalEeprom_WriteBuffer

#define phhalEeprom_WriteBuffer (   pbBuffer,
  pbEeprom,
  dwBytesToWrite 
)

This function writes data to EEPROM.

This API can be used to write data of any valid size(even mis-aligned) to any valid address in the EEPROM(even mis-aligned addresses).

Note
The Write is synchronous i.e. The API will return only after the write operation completes.
This API is not optimized for Page writes. phhalEeprom_WritePage() is a better choice when writing to a single page in the EEPROM.
Parameters
[in]pbBufferPointer to buffer to be copied - can be in RAM or EEPROM
[in]pbEepromPointer to EEPROM buffer where data will be copied
[in]dwBytesToWriteNumber of bytes to be written. Must not be zero.
Returns
Status of the Operation
Return values
PH_ERR_SUCCESSif EEPROM Write Operation completed successfully.
maskedPH_ERR_READ_WRITE_ERROR HW related errors
maskedPH_ERR_INVALID_PARAMETER if parameters passed are invalid and parameter checking is enabled in build.
maskedPHHAL_ERR_EEPROM_BUSY if another EEPROM write is already ongoing.

§ phhalEeprom_HwReset

#define phhalEeprom_HwReset ( )

This function resets the EEPROM controller hardware.

The purpose of this API is to reset the EEPROM controller. This is useful if a write program phase never completes and the write operation needs to be aborted. A typical use case would be to use this macro inside the watch dog timer handler, when a EEPROM write operation is ongoing.

Returns
Status of the Operation
Return values
PH_ERR_SUCCESSalways
